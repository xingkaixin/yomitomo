# Yomitomo 帮助手册

> 伴读 · 你的 AI 阅读伙伴

Yomitomo（读伴）是一款桌面端 AI 伴读应用。它不替你读文章，而是在你阅读时陪你思考、批注、讨论，帮你读得更深。你的所有数据——文章、批注、对话、API 密钥——全部存储在本地，不会上传到任何服务器。

## 快速开始

### 1. 首次启动

首次打开 Yomitomo，会看到一段欢迎文字，介绍伴读的理念。阅读完毕后，点击「进入 Yomitomo」进入应用。

### 2. 配置 AI 模型供应商

要使用 AI 伴读功能，需要先配置至少一个模型供应商：

1. 点击侧边栏「设置」
2. 选择「模型与路由」
3. 点击「添加供应商」，选择一个预设供应商（如 DeepSeek、OpenAI、Anthropic 等），或填写自定义 OpenAI 兼容 API
4. 填入 API Key 和 Base URL（预设供应商会自动填充 Base URL）
5. 选择模型，点击「测试连接」确认可用
6. 保存后，在下方「任务路由」中为三类任务分别指定供应商

### 3. 导入第一篇文章

1. 点击侧边栏「阅读库」
2. 点击右上角「+」按钮
3. 选择「添加网页」粘贴文章链接，或选择「ePub 电子书」导入本地文件
4. 导入完成后，点击文章卡片进入阅读器

## 应用界面

Yomitomo 的主界面由以下部分组成：

### 侧边栏

左侧导航栏包含四个主要区域：

| 图标 | 名称 | 功能 |
|------|------|------|
| 📖 | **阅读库** | 管理和浏览已导入的文章和电子书 |
| 🤖 | **助手** | 查看和管理 AI 阅读助手 |
| 📊 | **统计** | 查看阅读活动数据和趋势 |
| ⚙️ | **设置** | 配置供应商、快捷键等选项 |

侧边栏可以通过左上角按钮折叠/展开。底部有每日一言卡片和个人资料入口。

### 个人资料

点击侧边栏底部的头像，可以修改：

- **昵称**：在讨论中显示的名字
- **用户名**：你的唯一标识
- **头像**：上传图片或使用文字头像
- **批注颜色**：你的高亮和批注在阅读器中的显示颜色

## 阅读库

阅读库是你所有文章和电子书的管理中心。

### 导入文章

点击右上角「+」按钮，有两种导入方式：

#### 添加网页

1. 在弹窗中粘贴文章的网址
2. 点击「导入」，Yomitomo 会自动提取标题、作者、正文和图片
3. 导入成功后文章出现在阅读库中

支持大部分公开网页文章。对于需要 JavaScript 渲染的页面（如 Cloudflare 保护页面、微信公众号文章），Yomitomo 会自动使用内置浏览器渲染后提取。

#### 导入 ePub 电子书

1. 在弹窗中选择本地 `.epub` 文件（支持 EPUB 2/3 格式）
2. 文件大小上限为 80MB
3. Yomitomo 会提取封面、目录、章节等元信息

### 筛选与排序

阅读库顶部提供筛选和排序工具：

**筛选条件：**
- 全部
- 新添加（未开始阅读）
- 阅读中
- 已读完

**排序方式：**
- 最近阅读
- 最近添加
- 批注数量
- 讨论数量

支持搜索框按标题或作者关键词搜索。

### 文章卡片

每张卡片显示：
- 文章封面（根据文章信息自动生成配色）
- 标题和作者
- 阅读进度
- 批注和讨论数量

右键或点击卡片上的更多按钮可以删除文章。点击卡片进入阅读器。

## 阅读器

点击文章卡片后进入全屏阅读器。

### 阅读界面

阅读器提供干净的阅读体验：

- **正文区域**：文章内容以舒适的排版呈现
- **高亮标记**：你和 AI 助手的批注以不同颜色高亮显示在文中
- **批注连线**：高亮文本与侧边栏批注之间有视觉连线
- **目录导航**：ePub 电子书支持章节目录，显示每章的批注统计

### 阅读器设置

阅读器内可调整：
- 字体大小
- 页面宽度

### 文本选择操作

选中文本后，可通过快捷键执行操作：

| 快捷键 | 操作 | 说明 |
|--------|------|------|
| `C` | 复制 | 复制选中的文本 |
| `A` | 批注 | 对选中文本创建批注 |

这些快捷键可在「设置 > 快捷键」中自定义。

## 批注与讨论

批注是 Yomitomo 的核心功能，它让阅读不再是单向接收，而是一场有来有回的对话。

### 创建批注

1. 在阅读器中选中一段文本
2. 按快捷键 `A`（默认），或使用弹出的操作按钮
3. 输入你的想法、疑问或笔记
4. 批注会以你的个人颜色高亮显示

### 批注类型

每条批注会被分类为以下类型之一：

| 类型 | 说明 |
|------|------|
| **要点** (key_point) | 文章的核心论点或关键信息 |
| **假设** (assumption) | 作者的前提假设或隐含条件 |
| **概念** (concept) | 需要理解的术语或概念 |
| **疑问** (question) | 你的困惑或待探讨的问题 |
| **引用** (quote) | 值得记录的原文 |

### 阅读意图

创建批注时可以选择阅读意图，告诉 AI 助手你希望得到怎样的回应：

| 意图 | 说明 |
|------|------|
| **解释** (explain) | 请助手解释这段内容 |
| **拆解** (decompose) | 请助手分解论证结构 |
| **质疑** (challenge) | 请助手挑战这个观点 |
| **追问** (question) | 请助手提出追问 |
| **关联** (connect) | 请助手关联其他知识 |

### 讨论线程

每条批注下方是一个讨论线程：

- 你可以写评论，像在页边空白处写笔记一样
- 在评论中 `@助手名称` 可以让 AI 助手加入讨论
- AI 助手会基于文章上下文和批注内容回复你
- 疑问类批注有状态追踪：未解答、已解答、搁置

### 批注导航

阅读器侧边栏提供批注列表，支持：
- 按类型、助手、状态筛选
- 点击批注跳转到对应文本位置
- 前后批注快速跳转

## AI 伴读

### 专注伴读

「专注伴读」是 Yomitomo 最核心的 AI 功能——你选择一位或多位 AI 阅读助手，让它们和你一起精读文章的某个章节或段落。

#### 使用方式

1. 在阅读器中，打开专注伴读面板
2. 选择参与伴读的 AI 助手（可多选）
3. 选择要阅读的段落范围
4. 设置批注密度（低/中/高）
5. 启动伴读

#### 工作原理

- AI 助手会按段落逐步阅读，在关键位置生成批注
- 每位助手有各自的专长和视角，批注风格各不相同
- 助手会记住之前段落的内容，批注具有连贯性
- 批注生成过程是流式的，你可以实时看到助手的思考

#### 上下文感知

AI 助手在批注时会考虑：
- 当前段落的完整文本
- 你已有的批注和讨论
- 之前章节的阅读记忆（摘要、关键追踪）
- 文章目录和整体结构

#### 剧透策略

对于 ePub 电子书，可以设置 AI 助手可参考的内容范围：
- 仅当前选段
- 当前段落
- 当前章节
- 已读内容
- 全书

## 读后笔记

读完一篇文章后，Yomitomo 可以帮你整理一份结构化的读后笔记。

### 三步流程

读后笔记的生成分为三个步骤：

#### 第一步：阅读评估

AI 基于你的所有批注和讨论，生成一份阅读审议报告——梳理你在阅读过程中的发现、疑问和洞见。

#### 第二步：AI 提炼

基于审议报告和原始证据，AI 提炼出一份结构化的读后笔记草稿，包含：
- 分段标题与核心内容
- 证据引用（关联到你的原始批注）
- 每条证据标注类型（要点、假设、概念、疑问、引用）

#### 第三步：笔记审阅

选择一位或多位审阅助手对笔记草稿进行质量评估。每位审阅者会给出：
- **结论**：通过或修改
- **发现摘要**：总体评价
- **具体问题**：按严重程度（高/中/低）列出
- **认可的亮点**和**遗漏的角度**

你可以针对某位审阅者重新生成评审，结果会自动合并。

### 复制笔记

笔记生成完成后，可以一键复制 Markdown 格式的完整内容，粘贴到你的笔记工具中。

## AI 助手

Yomitomo 内置了两类 AI 助手：**批注助手**（用于伴读）和**审阅助手**（用于笔记审阅）。

### 批注助手

在阅读器中陪你精读、生成批注的助手：

| 助手 | 角色 | 擅长 |
|------|------|------|
| **林知微** | 页边同读者 | 安静地陪你读，在需要的地方澄清概念、补充背景 |
| **周砚** | 根因审读者 | 审查论证前提，追溯因果链条 |
| **许问渠** | 追问导师 | 把模糊的困惑转化为精确的追问 |
| **陈砚书** | 洞见整理员 | 提取可迁移的洞见和行动要点 |
| **沈清源** | 概念翻译员 | 解释术语，提供概念的背景脉络 |
| **顾行简** | 结构领航员 | 识别文章结构，标注每个段落的功能 |

### 审阅助手

在笔记审阅阶段对读后笔记进行质量把关的助手：

| 助手 | 角色 | 擅长 |
|------|------|------|
| **梁证言** | 证据司书 | 核实事实、验证证据链和引用 |
| **叶听澜** | 读者权益官 | 确保笔记保留了你的阅读者声音 |
| **唐简** | 终审编辑 | 压缩冗余，提升表达清晰度 |
| **何明衡** | 逻辑复核官 | 检测推理漏洞和逻辑矛盾 |
| **苏定白** | 风险审查员 | 标记过度概括和决策风险 |
| **夏归宁** | 行动校准师 | 确保行动项具体、可执行 |

### 管理助手

在侧边栏「助手」页面中，你可以：
- 查看所有助手的介绍和角色说明
- 启用或禁用某个助手
- 每位助手有独立的配色，在阅读器中以各自的颜色显示批注

## 阅读统计

侧边栏「统计」页面展示你的阅读活动数据。

### 数据概览

- **今日活动**：当天的阅读行为摘要
- **累计记录**：已记录天数、每周活跃天数、活跃峰值
- **下一目标**：向 7 天连续阅读目标的进度
- **总计**：导入文章数、讨论次数、批注条数、助手参与次数

### 活动图表

以折线图展示一段时间内的阅读趋势：
- 文章导入数
- 批注创建数
- 讨论评论数

可切换「已记录」和「近 21 天」两种时间范围。

### 活动日历

以热力图形式展示近 70 天的阅读活动，每天的活跃程度以不同状态标识。连续活跃 7 天会获得特殊标记。

### 阅读洞察

系统自动生成的阅读习惯分析，包括：
- 活动高峰日
- 批注深度与讨论频率的对比
- AI 助手参与情况

- OpenAI

## 常见问题

### AI 功能没有反应？

请确认：
1. 已在「设置 > 模型与路由」中添加至少一个供应商
2. 供应商的 API Key 填写正确，「测试连接」通过
3. 已在「任务路由」中为对应任务指定了供应商

### 网页文章导入失败？

可能的原因：
- 文章链接需要是 `http://` 或 `https://` 开头的有效网址
- 部分网站有反爬保护，Yomitomo 会尝试自动渲染，但仍有少数网站无法解析
- 导入超时（默认 15-20 秒），检查网络连接

### ePub 导入失败？

- 确认文件是标准的 `.epub` 格式（EPUB 2 或 EPUB 3）
- 文件大小不能超过 80MB

### 数据存储在哪里？

所有数据存储在本地应用数据目录中，不会上传到任何外部服务器。API 密钥加密存储。

### 支持哪些操作系统？

目前支持 macOS 和 Windows。
