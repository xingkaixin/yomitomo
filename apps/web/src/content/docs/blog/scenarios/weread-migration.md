微信读书是国内体验最流畅的电子书阅读产品之一。很多人用它读了几十本书，积累了几百条划线和想法。但迟早会撞上一个问题：**你的阅读数据被锁在平台里**。

微信读书不提供官方的全文导出功能。你想把自己的划线整理成读书笔记，只能一条条手动复制。你想给某条想法补写后续思考，只能在原 app 里追加评论，无法链接到其他书籍、也无法自由编辑格式。更关键的是，如果有一天你不再使用微信读书，那些划线和想法就变成了无法访问的「数字遗迹」。

Yomitomo 通过微信读书的开放 API，提供了一条**本地化迁移路径**：把你的划线、想法和阅读进度同步到本机，在本地继续深度加工。

## 第一步：获取 API KEY，建立连接

Yomitomo 不自建微信读书中介服务器。同步是通过微信读书的官方 API 完成的，你只需要在微信读书网页版获取一个 API KEY（类似许多第三方读书统计工具的做法）。

具体获取步骤见 Yomitomo 文档中的「获取微信读书 API KEY」指南。整个过程大约两分钟：登录网页版、打开开发者工具、找到请求头中的关键字段。拿到 API KEY 后，在 Yomitomo 的「微信读书」设置中填入即可建立连接。

这个连接是一次性的配置。配置完成后，你的阅读数据通过本地应用直接请求微信读书接口，不经过任何第三方服务器。

## 第二步：选择性同步

Yomitomo 不会一次性全量拉取你的微信读书数据。你可以在「微信读书」页面选择要同步的书籍，按需同步该书的划线、想法和阅读进度。

同步后的数据会进入 Yomitomo 的阅读库，和本地导入的 EPUB、PDF、网页文章放在一起管理。这意味着你可以：

- 在微信读书的划线上继续添加批注——Yomitomo 的五种批注类型（要点、假设、概念、疑问、引用）同样适用于已同步的内容
- 在微信读书原有想法下继续补充评论和追问
- 在讨论区 `@` AI 助手，让它们基于你的微信读书划线和想法加入讨论
- 把微信读书中的多本书的划线整合到沉淀窗口，形成跨书籍的知识整理

## 第三步：本地化之后的自由

数据到了本地，你就有了平台给不了的自由：

- **全文搜索**：在阅读库中按标题关键词搜索你同步过的所有书籍
- **跨书关联**：沉淀窗口中可以同时引用多本书的划线，做专题式的知识整合
- **阅读统计**：Yomitomo 的统计页会纳入微信读书的阅读数据，和本地阅读数据一起展示活动趋势
- **数据备份**：Yomitomo 支持 SQLite 数据库备份和还原，你的微信读书笔记和本地笔记一起被保护

同步不是一次性的「导出→结束」。你可以随时重新同步，更新最新的阅读进度和新增的划线。

## 适合谁

微信读书重度用户，已经积累了大量划线和想法，希望将这些阅读数据本地化保存、深度加工、不受平台限制地自由使用。
