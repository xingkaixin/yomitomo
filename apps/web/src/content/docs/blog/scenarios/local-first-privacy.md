在 2026 年，「云端优先」已经成为软件开发的默认选择。笔记在云端、文档在云端、阅读数据在云端。这个模式带来了跨设备同步的便利，但它也带来了一个被普遍接受却很少被讨论的代价：**你的阅读数据不再属于你**。

阅读数据和其他数据不同。它不是日历事件或任务清单——那些是你和世界交互的记录。阅读数据是你的**思考过程**：你在某段文字旁的批注、你对某个观点的质疑、你从一个概念联想到另一个概念的路径。这些数据构成了你的认知轨迹。把它交给云端，意味着你默认接受了服务商可以分析它、用他训练模型、或者在停止服务后让它消失。

Yomitomo 选择了另一条路：**本地优先**。

## 所有数据保存在你的电脑上

Yomitomo 的所有阅读数据——导入的文章、EPUB、PDF、你的划线、批注、讨论、沉淀稿、阅读统计——全部存储在桌面端应用数据目录中。没有 Yomitomo 云端服务器存储你的任何阅读内容。

这意味着：

- 你不需要注册账号。下载安装，直接使用
- 你的阅读数据不会因为服务商的政策变更而受影响
- 你可以完全离线使用所有功能（除 AI 助手需要联网调用模型 API 外）
- 没有人能分析你的阅读偏好和思考模式

## API Key 使用系统 keyring 保存

Yomitomo 需要连接 AI 模型供应商（OpenAI、Anthropic、DeepSeek 等）的 API 来实现助手功能。但你的 API Key 不会存储在 Yomitomo 的普通配置文件中。

Yomitomo 使用操作系统级别的 keyring（macOS Keychain、Windows Credential Manager）来存储 API Key。SQLite 数据库中只保留 provider 配置和 key 的引用，不保留明文 Key。这意味着即使有人获得了你的 Yomitomo 数据文件，也无法读取你的 API Key。

API 调用是从你的本地应用直接发送到模型供应商的服务器。Yomitomo 不在中间做任何代理或转发。你的 API Key 只有你和模型供应商知道。

## 完全离线可用

Yomitomo 的核心阅读功能——导入文章、阅读 EPUB/PDF、划线、批注、讨论、沉淀——全部离线可用。你不需要网络连接就可以完成一次完整的深度阅读。

AI 助手功能需要联网（因为模型推理在云端），但这是可选的。你可以完全不启用 AI 助手，把 Yomitomo 当作一个纯粹的离线阅读和批注工具使用。

这个设计与「云端优先」工具形成了鲜明对比：云端工具在网络断开时会降级甚至不可用，而本地优先工具在联网时**获得额外能力**，而不是断网时**失去核心能力**。

## 数据备份和迁移

本地优先不等于数据脆弱。Yomitomo 提供了 SQLite 数据库的备份和还原功能。你可以定期导出数据库文件，备份到外部硬盘或你信任的云存储中。

数据库格式是标准的 SQLite，这意味着即使有一天你不再使用 Yomitomo，你的数据也不会被锁定在专有格式中。你可以用任何 SQLite 工具打开、查询、导出你的阅读数据。

微信读书同步的数据同样存储在本地 SQLite 中。你从微信读书同步过来的划线和想法，一旦到了本地，就完全脱离了对微信读书平台的依赖。

## 阅读的私密性

阅读本质上是一件私密的事。你在某本书里标记的段落、写下的疑问，可能是你正在思考的问题、正在面对的困惑、甚至正在进行的自我对话。这些内容不适合出现在任何云端服务的分析仪表盘里。

本地优先保证了你可以放心地在阅读中暴露自己的无知和困惑——而这恰恰是深度学习的前提。如果你知道你的每一条批注都可能被分析、被训练模型、或者某天被泄露，你就会下意识地自我审查，写一些「安全的」、不会暴露弱点的笔记。而这种「安全笔记」是没有任何认知价值的。

## 适合谁

对数据隐私敏感、经常在无网络环境阅读、或者单纯认为「我的思考应该留在我自己的硬盘上」的人。如果你对「所有数据上云」的趋势感到不安，本地优先是你最根本的解决方案。
