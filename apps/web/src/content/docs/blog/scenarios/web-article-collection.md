每个知识工作者的浏览器里都有一座「稍后读坟场」。Pocket、Instapaper、浏览器书签栏、微信「文件传输助手」——你往里面塞了几百篇文章，怀着「迟早会读」的心情。偶尔打开一次，面对列表感到焦虑，划掉几篇标题，关上，继续往里塞。

问题不是「稍后读」这个概念不好，而是它把**收藏**当成了终点。你收藏一篇文章时，心里的叙事是「我将来会读」。但工具的设计让你只能「读或不读」——打开→扫一眼→归档。没有中间的加工状态：没有划线、没有笔记、没有产出。于是大部分文章读完之后和在收藏夹里没有本质区别——你不记得它说了什么，也不记得自己怎么看它。

Yomitomo 不做一个新的「稍后读」工具。它做的是把「收藏→扫一眼」的流程，升级为「导入→阅读→划线→沉淀」。

## 粘贴链接，开始阅读

在 Yomitomo 阅读库中点击「添加网页」，粘贴你刚看到的那篇文章的链接。Yomitomo 会提取标题、作者、正文和图片，生成一篇干净的阅读材料。

这一步替代了「存进稍后读」的动作。区别在于：存进稍后读意味着「迟点再说」，而导入到 Yomitomo 意味着「现在就读，或者计划读」。阅读库提供了按阅读状态筛选的功能——新添加、阅读中、已读完——你可以把导入当作一个轻量的阅读计划。

对于需要浏览器渲染的页面（比如重度 JavaScript 动态加载的文章），Yomitomo 会尝试使用内置浏览器渲染后再提取正文。如果仍然失败，少数有反爬保护的网站可能无法正常导入。

## 阅读时做标记，而不是「扫一眼」

打开导入的网页文章，你会进入和 EPUB、PDF 一样的阅读器。正文干净排版，两侧没有广告和推荐流。选中文本，按 `A`，选择批注类型，写下你的想法。

这个步骤是网页文章从「灰」变「活」的关键。你之前「扫一眼」的时候，大脑里的反应是「嗯，有道理」「这个数据有意思」「不太同意」——然后这些反应就蒸发了。现在你把这些反应变成了**文字记录**，贴在了对应的原文段落旁边。

你还可以把标记了「疑问」的段落丢给 AI 助手。`@许问渠` 会帮你把模糊的困惑转化成精确的问题，`@陈砚书` 会帮你从案例中提取可迁移的洞见。讨论落在那条划线旁边，成为你阅读轨迹的一部分。

## 从单篇文章到跨文章的知识整合

网页文章的特点是**短、散、多**。单篇 3000 字的长文、800 字的行业快讯、某个博客的技术分析——它们来自不同来源，但经常指向相同的主题。

Yomitomo 的沉淀窗口可以把多篇文章的划线聚合在一起。比如你最近读了五篇关于「AI 编程工具」的文章，每篇都画了几条线。在沉淀窗口中选择这五篇文章，把你分散的想法整合成一个结构化的沉淀稿：哪些工具的对比、哪些是你关心的维度、你的判断是什么。

单篇网页文章读完就归档的价值有限。但五篇相同主题的文章整合成一篇沉淀稿，就变成了一份**可复用的个人调研**。两个月后有人问你「那个 AI 编程工具怎么样」，你不用重新搜那五篇文章——你打开沉淀稿，里面有你的判断。

## 历史文章别弄丢

Yomitomo 可以开启「本地保存图片」选项。开启后，导入网页文章时会同时把正文中的图片保存到本机。这意味着即使原网站改版、删文、或者你断网，你仍然可以完整阅读这篇文章。

阅读库支持标题和作者关键词搜索。当你隐约记得「读过一篇讲 React Server Components 的文章」时，搜一下标题关键词就能找到。不会出现「我确定读过，但再也找不到了」的沮丧。

## 适合谁

每天会看到大量值得读的网页文章、但读完率不足 20% 的知识工作者。如果你厌倦了「收藏→遗忘→再收藏」的循环，想把碎片化阅读转变成可积累的知识资产。
