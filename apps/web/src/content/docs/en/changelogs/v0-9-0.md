Released: 2026-06-24

## Features

- The Library now shows web articles, ebooks, PDFs, WeRead items, and collections in one list. Type filters keep their session state, so returning from the reader lands closer to the view you left.
- Collections and pinned items are now available. A collection can hold local articles and WeRead items, opens as a filtered Library view, previews its contents on the cover, and lets items be removed from inside the collection.
- Multi-source add actions and empty states have been reorganized, while import completion, Library paging, and list transitions use more consistent motion.
- Distillation review now streams structured suggestions. Organized discussion lives in the draft area, and suggested additions, removals, or edits can be previewed in the draft before you accept them.
- WeRead supports manual or automatic sync, the app checks for updates silently with a persistent entry when an update is available, and model routing setup is clearer.
- Settings now include a switch for anonymous version and system metrics. It is used to understand active version and system distribution without collecting reading content, book titles, annotations, file paths, or AI conversations.

## Performance

- Related-passage retrieval reuses document-frequency statistics, reducing repeated work for long articles and books.
- SQLite storage now avoids duplicated ebook body and avatar data, with database cleanup to keep long-term growth more controlled.
- Chromium cache maintenance runs on startup, and logs now default to a 90-day retention window so local cache and log data do not grow without bound.

## Fixes

- Fixed web-article translation selection, reader focus navigation, PDF highlight click linking, and long ebook title display.
- Fixed failed assistant requests remaining in progress, WeRead settings save timers leaking, and pending ebook page totals appearing before calculation finishes.
- Unified reader tooltip timing, copy-icon swaps, inline annotation composer motion, page transitions, segmented controls, and App Lock reduced-motion behavior.
- The website now uses the refreshed brand mark, updated reader screenshots/help docs, the mobile table-of-contents entry, and stronger keyboard interactions.
