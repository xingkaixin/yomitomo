Released: 2026-06-08

## New

- Added desktop language switching, bilingual website docs, and localized assistant persona presentations.
- Added persistent reader chat so conversations around the current article, EPUB, or PDF can stay available while reading.
- Added in-reader full-text search for quickly finding keywords inside the reading surface.
- Unified reader actions into a floating toolbar and redesigned the article header, empty notes state, and annotation layout for constrained space.
- Deepened annotation and discussion flows with root-assistant reply routing and more actionable review proposals in distillation.
- Reworked Settings information architecture, surfaced assistant usage in stats, and added Library content-source customization, source-aware dialogs, and direction-aware transitions.
- Added release-notes dialogs to the desktop update flow, using website notes before update and bundled local notes after update.
- Added local annotation writes, rebuilt PDF and web article media covers, added ebook import cover feedback, and introduced the dusk indigo theme.

## Performance

- Reused more precomputed reading memory, EPUB index, and assistant summary data to reduce repeated scans and aggregation.
- Reworked article repository batch operations, import lifecycle, and image inlining orchestration to reduce blocking and failure surfaces during imports.
- Moved assistant runtime, model pricing, WeRead sync, and annotation memory flows toward Effect orchestration with clearer error boundaries.

## Fixes

- Fixed visual issues across Settings and Reader, including settings surfaces, menu layout, titlebar text constraints, reader titlebar polish, and annotation card details.
- Fixed Library list cards, web cover toolbar overlays, flat-cover progress bars, and PDF cover ratios.
- Fixed discussion avatar spacing, the add-thought composer shell, reply-free thought placement on open, and thought auto-scroll scope.
- Fixed SQLite native root isolation, article summary count scope, data recovery failures, and provider keyring regression coverage.
- Fixed website carousel behavior that could scroll the landing page horizontally.

## Engineering

- Split desktop main, renderer, source reader, settings, PDFium, reader shell, and global CSS boundaries to lower single-file complexity.
- Split AI assistant runtime, provider, context, reading memory, and shared/core domain modules, with export coverage tests.
- Added tests for data management, store-db recovery, article import workers, provider secrets, reader chat, and reader shell state.
- Added native root verification, a PR template, output-directory ignore rules, and pnpm/dependency updates.
