发布日期：2026-05-21

## 新功能

- 桌面端新增应用更新流程，并把关于页、引导页和官网下载入口连到 GitHub Release 产物。
- 新增产品官网，提供 Yomitomo landing page、macOS / Windows 下载链接、SEO 元信息、sitemap、robots 和社交预览图。
- 设置页新增数据管理能力，可查看数据目录、日志和数据库文件，备份或还原 SQLite 数据库，并在数据库版本不兼容时给出明确提示。
- Provider 设置改为安全保存 API key，支持显式查看已保存 key、拉取模型列表、配置任务路由，并简化模型设置表单。
- 阅读器想法和讨论体验升级，支持选区 `@助手` 路由、审阅助手评论、批注讨论卡、悬浮目录、双侧笔记栏和待处理助手状态。

## 调整

- 移除读后输出流程，让产品重心收敛到原文锚点、想法线程、评论讨论和助手共读。
- 阅读库拆分网页文章和 EPUB 电子书入口，优化首页布局、返回来源、资料卡片和 EPUB 导入记录体积。
- 阅读器布局和交互进一步打磨，包括源阅读器布局、批注连接线、想法计数、共读控制、头像 hover 和想法输入器交互。

## 性能

- EPUB 阅读复用 DOM 文本索引，减少翻页、页面更新和批注渲染的重复计算。
- 优化批注热路径、运行时 import 热点和阅读库 article payload 加载，降低大书和大库场景下的启动与交互成本。

## 修复

- 修复 EPUB 批注渲染、翻页速度、页面位置恢复、页面更新和堆叠批注连接线稳定性问题。
- 修复 Provider 删除 API key 后名称保留、开发数据库迁移历史保留、数据库兼容错误展示和目标批注归属保持问题。
- 修复阅读库返回后来源丢失、助手待处理态不可见、想法输入器交互细节和共读控制状态问题。

## 工程

- 拆分 main app state、source bookcase、设置面板、reader 组件、批注评论输入器、助手批注队列、agent runtime、provider settings 和 EPUB runtime 边界，降低核心 UI 与运行时代码复杂度。
- 抽取阅读卡片工作流状态、source agent 请求管线和代码健康热点边界，并补充运行时性能热点审计文档。
