发布日期：2026-06-01

## 新功能

- 新增阅读记忆底座，把段落、选区、线程和批注记忆写入本地存储，并在后续助手生成中复用已有判断链路。
- AI 助手运行时升级为可追踪的工具调用链路，支持选区动作、线程回复、共读工具循环、调用诊断视图，并将深度运行时迁移到 AI SDK。
- 批注讨论升级为独立讨论窗口和沉淀流程，支持助手参与讨论、沉淀稿整理、审阅助手反馈、发布沉淀卡片和沉淀统计。
- 新增主题契约、主题选择器和阅读器纸张背景控制，统一网页文章、EPUB 和 PDF 的阅读视觉；暗色纸张下 PDF 会保留原始页面颜色。
- 阅读库和导入体验支持网页文章封面与进度、批量文件导入、导入对话框打磨、卡片统计提示和 PDF/EPUB 显示元数据清理。

## 性能

- 缓存 lexical related passages，减少长文和长书共读时的重复关联段落检索。
- 优化文章导入流程，避免导入期间阻塞 UI 响应。
- 跳过重复的批注记忆回填，避免启动后反复执行同一批历史数据迁移。

## 修复

- 修复 PDF 批注锚点、列表文本溢出、PDF 暗色纸张颜色、EPUB 方向键翻页和阅读器 header 拖拽回归。
- 修复阅读记忆和助手运行时中的重复想法、线程记忆来源和工具调用稳定性问题。
- 修复阅读库分页大小持久化、微信读书详情封面、电子书章节笔记计数、电子书 header 封面 hover 和批注元数据隐式推断问题。
- 打磨批注卡片、助手头像、批注动作、快捷键 tooltip、封面 hover、关于页、主题纸张和通用 UI 交互。

## 工程

- 启用 type-aware lint 并更新依赖，抽取 selection context 工具，迁移深度运行时到 AI SDK，减少运行时边界重复逻辑。
- 移除 focus co-reading 旧流程，并裁剪桌面端打包产物中的 native module 文件，降低发布包冗余。
- 更新用户文档、官网下载区和第三方授权清单，确保官网版本入口与桌面端内置开源许可证信息一致。
