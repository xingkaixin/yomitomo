发布日期：2026-07-20

## 新功能

- 新增「沉淀库」：把全书已发布的沉淀汇总到一个可搜索的只读列表，支持全文搜索、显示来源上下文，并可一键跳回原始批注。(#663, #665)
- 电子书阅读器新增章节级双语阅读，在原文旁按章渲染缓存的译文，复用现有翻译供应商，并在重启后自动恢复。(#664)
- 合集卡片重做为「展开成扇」封面：最近的封面像手牌一样叠放，hover 或 focus 时扇形展开，让合集里不同来源的内容一目了然。(#672)
- 精修全应用的反馈动画：阅读进度改在合成层渲染，滚动更顺滑；设置保存反馈由过大的弹跳收敛为克制的入场；首次使用的空状态用柔和渐入引导。(#666)

## 性能

- 恢复分页目录，启动不再一次性加载整个书库，改用统一索引与 trigram 搜索，大书库依然流畅。(#651)
- 微信读书同步改用参数安全的批量明细插入，大合集成员改为线性追加，避免每次添加都整体拷贝。(#657, #655)
- 减少大批注栏、EPUB 锚点解析和阅读上下文范围查找的计算开销。(#654, #653, #652)

## 修复

- 保持合集导航同步，切换作用域时不再短暂显示上一个合集的内容。(#673)
- 恢复阅读库的本地化导入错误详情与卡片菜单状态，菜单触发按钮不再因焦点转移而消失。(#667)
- 恢复设置的草稿变更检测，让新增设置项也能保存，并用失败时的确切值重试保存。(#674)
- 阅读进度写入改为串行，IPC 失败后保留未保存进度待重试，进度不再丢失。(#648)
- 微信读书同步在应用变更前要求权威数据，残缺响应不再覆盖本地笔记。(#647)
- PIN 验证加入统一冷却节流，抵御暴力尝试。(#649)
- 供应商与定价的外部 JSON 响应在使用前先做校验，并修补有漏洞的生产依赖。(#671, #658)

## 工程

- 将 workspace 迁移到 Effect v4，强制 v4 运行时语义并保留类型化失败。(#668, #669, #670)
- 加固 favicon 抓取、协调凭据删除，并在阅读器写入路径统一使用最新文章快照与等待批注持久化。(#646, #650, #645, #644)
- 重整阅读器、AI、批注、微信读书、IPC 与桌面 store 的模块边界，明确归属并收窄写入。(#643, #659–#662, #675–#703)
