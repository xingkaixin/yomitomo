# Yomitomo 隐私说明

最后更新：2026-05-10

Yomitomo 是一个用于网页文章阅读、批注和本机阅读库同步的浏览器扩展。本说明解释 Yomitomo 扩展端处理哪些数据、这些数据用于什么目的，以及用户如何管理这些数据。

## 单一用途

Yomitomo 的单一用途是把用户主动选择的网页文章转换为阅读器视图，并支持高亮、批注、评论和发送到本机 Yomitomo 阅读库。

## 处理的数据

当用户主动进入阅读器模式或发送文章到阅读库时，Yomitomo 会处理当前网页中的以下信息：

- 页面标题、网址、规范链接和站点信息。
- 文章正文、作者、摘要、发布时间、封面图和站点图标。
- 用户创建的高亮、批注、评论和阅读状态。
- 与本机 Yomitomo 桌面端配对所需的本地连接状态和配对标识。

Yomitomo 使用这些数据提供阅读器展示、文章保存、批注管理、评论讨论和本机桌面端同步能力。

## 本地存储

Yomitomo 使用 Chrome 扩展本地存储保存必要状态，包括配对信息、文章记录、批注数据和界面状态。这些数据保存在用户浏览器环境中，用于恢复阅读和同步状态。

## 本机桌面端通信

当用户已安装并运行 Yomitomo 桌面端，且完成配对后，扩展会通过 `127.0.0.1` 上的本机 WebSocket 连接与桌面端通信。该连接用于把用户选择的文章、批注和评论发送到本机阅读库，并同步本机阅读状态。

## 第三方共享

Yomitomo 扩展端仅将用户数据用于提供上述阅读、批注和本机同步功能。扩展端的数据处理不用于广告画像、广告投放、信用评估或数据经纪。

如果用户在 Yomitomo 桌面端配置了 AI provider，桌面端可能会按用户操作把文章内容、选中文本、批注或评论发送给用户配置的 provider 以生成回复、批注或读后内容。该行为由桌面端配置和用户操作触发，并受对应 provider 的服务条款和隐私政策约束。

## 权限用途

Yomitomo 扩展使用以下 Chrome 权限：

- `storage`：保存扩展状态、配对信息、文章记录和批注数据。
- `scripting`：在用户主动点击时向当前标签页注入内容脚本，打开阅读器模式或提取文章正文。
- `<all_urls>`：让用户可以在不同网站的文章页使用阅读器和批注功能。
- `http://127.0.0.1/*` 和 `ws://127.0.0.1/*`：连接用户本机运行的 Yomitomo 桌面端，用于发送文章和同步批注。

## 用户控制

用户可以在浏览器扩展管理页面停用或移除 Yomitomo。用户也可以清除浏览器扩展本地存储中的 Yomitomo 数据，或在 Yomitomo 桌面端管理已同步到本机阅读库的文章和批注。

## Chrome Web Store 用户数据政策

Yomitomo 对通过 Google API 获得的信息的使用遵守 Chrome Web Store 用户数据政策，包括 Limited Use 要求。
